# HOI4 Agent Skill Implementation Plan

## Goal

Create the `0.1.0` MVP for a fully neutral HOI4 modding agent skill. The MVP should be instruction-only, auditable, platform-portable, and focused on safe agent behavior for Hearts of Iron IV mod repositories.

## Phase 1: Root Project Files

Create:

- `README.md`
- `LICENSE`
- `SECURITY.md`
- `CONTRIBUTING.md`

Requirements:

- README positions the project as a portable agent skill, not a Codex-only package.
- README lists supported formats: Codex Skills, Claude Skills, Cursor Rules, Cline Rules, and `AGENTS.md`.
- README clearly states that the repo does not include copied vanilla HOI4 files, DLC files, Paradox artwork, icons, or proprietary assets.
- SECURITY states the skill is instruction-only by default and should not execute commands, download files, modify files outside the active mod repo, or override platform safety instructions.
- CONTRIBUTING explains that adapters must stay thin and canonical behavior belongs in `skills/hoi4-modding/SKILL.md`.

## Phase 2: Canonical Skill

Create:

- `skills/hoi4-modding/SKILL.md`
- `skills/hoi4-modding/LICENSE.txt`

Requirements:

- `SKILL.md` includes YAML frontmatter with `name`, `description`, `license`, and neutral metadata.
- It uses platform-neutral language.
- It defines the operating model, first actions, reference-loading policy, editing checklist, debugging workflow, and response format.
- It instructs agents to load detailed references only when relevant.
- It avoids copied HOI4 examples that would constitute proprietary game content.

## Phase 3: Core References

Create:

- `skills/hoi4-modding/references/01-folder-structure.md`
- `skills/hoi4-modding/references/02-paradox-script.md`
- `skills/hoi4-modding/references/03-focus-trees.md`
- `skills/hoi4-modding/references/04-events.md`
- `skills/hoi4-modding/references/05-decisions.md`
- `skills/hoi4-modding/references/06-ideas-and-modifiers.md`
- `skills/hoi4-modding/references/07-countries-history-states.md`
- `skills/hoi4-modding/references/08-localization.md`
- `skills/hoi4-modding/references/09-gfx-interface-icons.md`
- `skills/hoi4-modding/references/10-scripted-effects-triggers.md`
- `skills/hoi4-modding/references/11-debugging-error-log.md`
- `skills/hoi4-modding/references/13-common-failure-modes.md`

Requirements:

- Keep references concise and procedural.
- Focus on agent behavior and validation, not exhaustive HOI4 documentation.
- Include small original skeleton snippets where useful.
- Do not include copied vanilla files, game text, DLC content, or Paradox assets.

## Phase 4: Platform Adapters

Create:

- `adapters/codex/README.md`
- `adapters/claude/README.md`
- `adapters/cursor/AGENTS.md`
- `adapters/cursor/.cursor/rules/hoi4-modding.mdc`
- `adapters/cline/AGENTS.md`
- `adapters/cline/.clinerules/hoi4-modding.md`

Requirements:

- Adapters are thin.
- Adapters point to the canonical `skills/hoi4-modding/` directory.
- Cursor and Cline rule files contain a compact behavior summary only.
- No adapter becomes a fork of the full skill.

## Phase 5: Prompts And Smoke Tests

Create:

- `prompts/debug-error-log.md`
- `prompts/create-focus-branch.md`
- `prompts/add-event-chain.md`
- `prompts/review-mod-pr.md`
- `tests/expected-agent-behavior.md`
- `tests/smoke-prompts.md`

Requirements:

- Prompts demonstrate realistic agent requests without including proprietary game content.
- Expected behavior tests describe what a good agent should inspect, change, validate, and report.
- Smoke prompts cover focus trees, event chains, localization, `error.log`, and review behavior.

## Phase 6: Review

Verify:

- canonical skill is platform-neutral
- adapters are thin
- references are concise
- no scripts or dependency installs exist
- no proprietary game content is included
- README installation guidance is clear
- security posture is explicit

## Completion Criteria

The MVP is complete when all planned files exist, the root README explains how to use the skill across supported platforms, and the skill can be installed or adapted without requiring scripts, binaries, external downloads, or platform-specific behavior.
